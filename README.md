# Human Proteome Exhaustive Mutagenesis

## Overview
<p>
This repository contains the computational pipeline used to generate a proteome-wide exhaustive missense variant effect prediction resource for the human reference proteome.

The dataset covers all proteins in the UniProt human reference proteome (Proteome ID: **UP000005640**) and includes all 19 possible amino acid substitutions at every protein position, resulting in a total of **564,570,641** missense variants across **20,659** proteins.

Predictions were generated using the complete QAFI predictor family together with several widely used external variant effect predictors.

The complete prediction resource is freely available on [Zenodo](DOI).

The QAFI methodology is described in the [QAFI publication](link).

## Dataset

The released dataset is organized as one directory per protein.

Each directory contains a single file:

```text
<UniProtID>/
└── <UniProtID>_variants.tsv
```

Each file contains exhaustive predictions for every possible missense substitution in the corresponding protein.

The dataset includes predictions generated by the QAFI family together with external predictors used throughout this work, including:

* AlphaMissense
* popEVE
* EVE
* CPT-1
* REVEL
* BayesDel
* QAFI family models

Coverage varies across proteins depending on the availability of structural models, multiple sequence alignments, and other required input features.

---

## Computational pipeline

The complete workflow consists of the following stages:

1. Generate all possible missense variants for every protein in the human reference proteome.
2. Integrate predictions from external variant effect predictors.
3. Compute predictions using the complete QAFI model family.
4. Merge all prediction scores into unified per-protein variant tables.
5. Perform quality control and coverage analyses.
6. Generate the final release dataset.

---

## Analysis utilities

The `analysis/` directory contains scripts used to:

* compute proteome-wide coverage statistics;
* generate dataset manifests;
* audit the final release;
* format prediction tables.

These scripts are intended for quality control and reproducibility and are not required to use the released dataset.
